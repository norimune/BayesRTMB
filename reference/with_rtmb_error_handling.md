# Catch and Translate RTMB Errors into User-Friendly Messages

This internal helper function wraps an R expression in a \[tryCatch()\]
block to capture cryptic error messages generated by the RTMB automatic
differentiation engine or C++ backend, translating them into descriptive
hints in Japanese.

## Usage

``` r
with_rtmb_error_handling(expr, block_name)
```

## Arguments

- expr:

  An R expression to be evaluated, typically representing the logic
  within a specific model block.

- block_name:

  A character string identifying the context of the error (e.g.,
  "parameters", "model", "transform", or "generate").

## Value

The result of evaluating the `expr`. If an error occurs, it throws a
formatted \[stop()\] message.

## Details

It uses regular expressions to categorize common RTMB failure modes,
such as:

- \*\*Missing Objects/Functions\*\*: Detects if a variable or a specific
  distribution function (e.g., \`normal_lpdf\`) is missing from the
  environment.

- \*\*Index Errors\*\*: Identifies subscript out of bounds errors.

- \*\*AD Type Destruction\*\*: Catches the "lost class attribute" error,
  often caused by incorrect initialization of vectors (e.g., using
  \`numeric(N)\` and partial assignment inside loops).

- \*\*Matrix Dimension Mismatch\*\*: Identifies non-conformable
  arguments in matrix multiplication.
